# 캐스팅엔 소싱팀 문서 자동화 도구

## 역할
너는 캐스팅엔 소싱팀을 위한 문서 자동화 Streamlit 앱을 만드는 풀스택 개발자다.
소싱팀은 매일 ① 고객사 구매 요청을 입찰공고문으로 변환하고 ② 공급업체 계약서의 위험 조항을 검토한다.
이 두 작업을 하나의 앱에서 처리할 수 있도록 탭 구조로 구현한다.

## 앱 개요
- 탭 1: 입찰공고문 자동 생성기
- 탭 2: 계약서 리스크 분석기
- 파일명: app.py
- 실행 위치: Castingn-AI/9.소싱문서자동화/code/

## 기술 스택
- streamlit
- openai (GPT-4o-mini)
- python-docx
- PyPDF2 또는 pdfplumber (계약서 PDF 텍스트 추출)
- python-dotenv

## 프로젝트 구조
```
Castingn-AI/9.소싱문서자동화/
├── code/
│   └── app.py
├── prompts/
│   └── 소싱문서자동화.txt
└── data/
```

## 환경 변수
- .env 파일 위치: Castingn-AI/.env
- OPENAI_API_KEY 필수
- load_dotenv()로 절대경로 기준 로드

# 탭 1: 입찰공고문 자동 생성기

## 배경
소싱팀은 고객사로부터 카카오톡/슬랙/이메일로 비공식 구매 요청을 받는다.
예시: "사무용 의자 500개요. 예산 3천만. 3주 안에 납품. 등받이 조절 되는 거로."
이를 공식 입찰공고문 Word 파일로 변환하고, 발송할 업체 이메일 초안까지 생성한다.

## 입력 UI
- 구매 요청 원문 텍스트 입력 (text_area, placeholder: "카톡/슬랙/메일에서 받은 구매 요청 내용을 그대로 붙여넣으세요")
- 고객사명 (text_input)
- 카테고리 선택 (selectbox): 사무용품/IT장비/청소용역/물류/시설관리/소모품/기타
- 발송 업체 수 (slider, 3~10개)
- [입찰공고문 생성] 버튼

## GPT 처리 로직

### 시스템 프롬프트
```
당신은 기업 구매 전문가입니다. 비공식 구매 요청 텍스트를 분석하여 공식 입찰공고문과 업체 발송 이메일을 생성합니다.
반드시 JSON 형식으로 응답하세요.
```

### 사용자 프롬프트
```
다음 구매 요청을 분석하여 입찰공고문과 업체 발송 이메일을 생성해주세요.

고객사: {고객사명}
카테고리: {카테고리}
구매 요청 내용: {원문 텍스트}

다음 JSON 형식으로 응답하세요:
{{
  "공고문": {{
    "제목": "",
    "입찰개요": "",
    "품목명": "",
    "규격_및_사양": "",
    "수량": "",
    "납품기한": "",
    "예산": "",
    "평가기준": ["가격 (40%)", "납기 준수 능력 (30%)", "품질 및 AS (30%)"],
    "제출서류": ["견적서", "사업자등록증 사본", "회사소개서"],
    "제출기한": "",
    "문의처": "캐스팅엔 소싱팀",
    "누락항목_경고": []
  }},
  "업체이메일": {{
    "제목": "",
    "본문": ""
  }}
}}
```

## 출력 UI
1. 공고문 미리보기 (expander로 펼치기)
   - 누락 항목 경고 (빨간 경고박스)
2. 업체 이메일 초안 미리보기
3. [Word 파일 다운로드] 버튼

## Word 파일 생성
- python-docx로 입찰공고문 작성
- 제목: 굵게, 16pt
- 섹션별 구분선 포함
- 파일명: f"{고객사명}_입찰공고문_{오늘날짜}.docx"
- st.download_button으로 다운로드

# 탭 2: 계약서 리스크 분석기

## 배경
소싱팀은 연간 수천 건의 공급업체 계약서를 검토한다.
법무팀 없이 계약서 PDF를 업로드하면 AI가 위험 조항을 자동 탐지하고 수정 요청 초안을 생성한다.

## 탐지 리스크 유형
- 🔴 HIGH: 납품 지연 면책 조항, 손해배상 한도 제한, 자동갱신 조항, 일방적 계약 해지권
- 🟡 MEDIUM: 가격 변동 허용 조항, 분쟁 관할법원 (공급업체 유리), 지식재산권 귀속 불명확
- 🟢 LOW: 납품 방식 미명시, 검수 기준 미상세, 하자보증 기간 단기

## 입력 UI
- 계약서 PDF 업로드 (file_uploader)
- 공급업체명 (text_input)
- 계약 유형 선택 (selectbox): 물품공급/용역/임대/유지보수
- [리스크 분석 시작] 버튼

## PDF 처리 로직
```python
import pdfplumber

def extract_text_from_pdf(uploaded_file):
    with pdfplumber.open(uploaded_file) as pdf:
        text = ""
        for page in pdf.pages:
            text += page.extract_text() or ""
    return text
```

## GPT 처리 로직

### 시스템 프롬프트
```
당신은 기업 구매 계약 전문가입니다. 계약서 텍스트에서 구매자(갑)에게 불리한 조항을 탐지하고 수정 요청 초안을 작성합니다.
반드시 JSON 형식으로 응답하세요.
```

### 사용자 프롬프트
```
다음 계약서를 분석하여 리스크 조항을 탐지해주세요.

공급업체: {공급업체명}
계약 유형: {계약유형}
계약서 내용:
{추출된 텍스트}

다음 JSON 형식으로 응답하세요:
{{
  "계약서_요약": "",
  "리스크_목록": [
    {{
      "조항명": "",
      "원문": "",
      "리스크_유형": "HIGH/MEDIUM/LOW",
      "문제점": "",
      "수정_요청_초안": ""
    }}
  ],
  "종합_의견": "",
  "계약_체결_권고": "권고/조건부권고/보류"
}}
```

## 출력 UI
1. 종합 리스크 요약 (메트릭: HIGH 건수 / MEDIUM 건수 / LOW 건수)
2. 계약 체결 권고 상태 (🔴 보류 / 🟡 조건부권고 / 🟢 권고)
3. 리스크 목록 (expander로 각 조항 펼치기)
   - 원문 인용
   - 문제점 설명
   - 수정 요청 초안 (복사 버튼)
4. [리스크 분석 보고서 다운로드] 버튼 (Word)

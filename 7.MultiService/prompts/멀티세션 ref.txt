7. 멀티세션 저장기능

#목적

# 상세기능
- Supabase 을 활용해서 세션 저장/로드 기능 추가
- 자동 저장: 파일 처리 및 대화 후 자동 세션 저장
- 세션 제목 자동 생성: LLM 기반 제목 생성합니다. 첫번째 질문과 답변을 요약해서 세션명을 만듭니다.
- 멀티선택 바에서 선택을 하면 자동으로 세션을 로드를 해주세요.
- 만일 streamlit 을 중지하고 다시 시작해도 이전 대화는 모두 저장해서 다시 불러 사용할 수 있도록 해주세요.
- vector database도 supabase를 사용해주세요. 따라서 이미 embedding이 된 내용은 다시 사용할 수 있도록 해주세요.
- 이때 embedding은 openai 를 사용해주세요.
- 문서에 대해서 질문에 대한 답변을 할 때, 향후 더 필요한 질문3개를 만들어서 답변 마지막에 추가해주세요.
- 질문을 하면 답변을 모두 완료될때까지 기다려야 하는데, 이것을 stream mode로 변경시켜줘. 즉 내가 질문하면 바로 생성된 답변이 display되도록 하도록 해줘.

# 오류 수정 내역
## 1. file_name NULL 오류 수정
- 문제: PDF 처리 시 vector_documents 테이블의 file_name 컬럼에 NULL 값이 저장되어 NOT NULL 제약 조건 위반 오류 발생
- 원인: SupabaseVectorStore.from_documents()가 메타데이터의 file_name을 file_name 컬럼에 자동 매핑하지 않음
- 해결 방법:
  * 각 PDF 파일을 개별적으로 처리하여 분할된 문서에 파일명을 명시적으로 추가
  * SupabaseVectorStore 대신 Supabase에 직접 저장하여 file_name, session_id, embedding 등을 포함
  * 배치 처리로 성능 유지 (배치 크기: 10)
  * 저장 후 SupabaseVectorStore를 검색용으로만 초기화

## 2. RAG 검색 필터 오류 수정
- 문제: RAG 답변 생성 시 'SyncRPCFilterRequestBuilder' object has no attribute 'params' 오류 발생
- 원인: SupabaseVectorStore.as_retriever()의 search_kwargs에 filter를 전달하는 방식이 지원되지 않음
- 해결 방법:
  * Supabase RPC 함수(match_vector_documents)를 직접 호출하여 세션 ID로 필터링
  * OpenAI Embeddings로 쿼리 embedding 생성 후 Supabase에 직접 전달
  * Supabase 응답을 LangChain Document 객체로 변환
  * RPC 호출 실패 시 일반 retriever 사용 후 필터링하는 대체 방법 제공

# 역할
너는 음성 파일 기반 회의록 자동 생성 시스템을 만드는 풀스택 개발자다.
Streamlit 웹 앱을 구축하고, OpenAI Whisper API로 STT 변환 후
GPT로 회의록을 정리하여 Word 템플릿에 매핑하는 Python 프로그램을 작성한다.

# 현재 데이터 상태

이미 다음 파일이 존재한다:

- ./8.STT/회의록 양식.docx

템플릿 파일 구조:
- {{회의명}}: 회의 제목
- {{부서}}: 담당 부서
- {{작성자}}: 회의록 작성자
- {{회의일시}}: 회의 날짜 및 시간
- {{회의장소}}: 회의 장소
- {{회의안건}}: 회의 안건
- {{참석자}}: 참석 인원
- {{회의내용}}: GPT가 생성한 회의 내용

# 폴더 구조 (고정)

- ./8.STT/prompts/     → 프롬프트 저장 위치
- ./8.STT/회의록 양식.docx → 회의록 템플릿 파일

# 최종 산출물

- ./8.STT/code/app.py 생성
- Streamlit 웹 앱 실행 시 회의록 자동 생성 기능 제공

# 프로그램 파일명

app.py

# 필요 라이브러리

- streamlit: 웹 앱 UI 구축
- openai: Whisper API (STT) 및 GPT API 호출
- python-docx: Word 템플릿 읽기/쓰기
- python-dotenv: .env 파일에서 환경 변수 로드
- pydub: 오디오 파일 변환 (ffmpeg 필요)
- requests: OpenAI API 직접 호출

# 외부 의존성

- ffmpeg: pydub 오디오 변환에 필요
  - 설치: https://www.gyan.dev/ffmpeg/builds/ 에서 다운로드
  - ffmpeg-release-essentials.zip 다운로드 후 bin 폴더를 시스템 PATH에 추가

# 전체 작업 흐름

1. Streamlit 페이지 설정
   - 페이지 제목: "회의록 자동 생성"
   - 페이지 아이콘: 📝
   - 레이아웃: wide

2. 환경 변수 설정
   - LLM 사용을 위한 API KEY는 .env에 있으니 반드시 python-dotenv의 load_dotenv()를 사용해 로드하세요.
   - .env 경로는 AI-Project/.env 입니다. 실행 위치와 무관하게 절대경로 기준으로 명시적으로 로드하세요.
   - 환경변수가 없으면 예외로 종료하지 말고 사용자에게 키 누락 메시지를 보여주세요.
   - API 키가 없으면 에러 메시지 표시

3. 템플릿 파일 경로 찾기
   - __file__ 기준으로 상위 폴더 검색
   - 현재 작업 디렉토리 기준 검색
   - 파일을 찾으면 성공 메시지, 못 찾으면 에러 메시지

4. 회의 정보 입력 폼
   - 회의명 (필수)
   - 작성자 (필수)
   - 회의 일시 (필수)
   - 회의 장소 (필수)
   - 부서 (필수)
   - 참석자 명단 (필수)
   - 회의 안건 (필수)

5. 음성 파일 업로드
   - 지원 형식: mp3, wav, m4a, mp4, webm, flac, mpeg, mpga, oga, ogg
   - 파일 업로드 후 미리보기 제공
   - 파일명 및 파일 크기 표시

6. STT 변환 (1단계)
   - MP3 파일이면 변환 없이 바로 Whisper API로 전송
   - MP3가 아니면 pydub로 MP3로 변환 (ffmpeg 사용)
   - OpenAI Whisper API로 텍스트 변환 (requests 직접 호출)
   - 변환된 텍스트 세션 상태에 저장
   - 사용자가 STT 결과를 직접 수정할 수 있도록 text_area 제공

7. 회의록 생성 (2단계)
   - GPT-4o-mini 모델 사용
   - JSON 형식으로 응답 받기 (response_format: json_object)
   - 회의내용만 생성 (메모사항 없음)
   - 불릿 리스트 형식으로 깔끔하게 정리

8. 최종 문서 생성 (3단계)
   - 템플릿의 플레이스홀더를 실제 데이터로 교체
   - DOCX 파일 다운로드 제공

# GPT 프롬프트 설계

## 시스템 프롬프트
"당신은 전문적인 회의록 작성자입니다. 회의 내용을 깔끔한 불릿 리스트 형식으로 정리합니다. 반드시 JSON 형식으로 응답하세요."

## 사용자 프롬프트 구조
- 회의 정보 (회의명, 회의안건, 참석인원)
- 변환된 음성 내용
- JSON 응답 형식 지정:
  {
      "회의내용": "반드시 하나의 문자열로 작성 (리스트 금지). 줄바꿈은 \\n 사용"
  }

## 중요 - 회의내용 형식
- GPT 응답의 "회의내용"은 반드시 문자열(string)이어야 함
- 리스트나 배열로 반환하면 안 됨
- 줄바꿈이 필요한 경우 \\n 사용
- 만약 리스트로 반환되면 format_content_to_string() 함수로 문자열 변환 처리

## 회의내용 작성 규칙
1. 회의 개요
   • 회의 목적과 배경 설명

2. 주요 논의 사항
   • 첫 번째 논의 사항
     - 세부 내용 및 의견
     - 결정 사항

3. 결정 사항
   • 결정된 사항 목록

4. 향후 계획
   • 후속 조치 사항

# STT 변환 함수 (transcribe_audio)

주요 로직:
1. 업로드된 파일 내용을 임시 파일로 저장
2. pydub.AudioSegment로 오디오 로드
3. MP3 형식으로 변환하여 저장
4. requests로 OpenAI Whisper API 직접 호출
5. 한국어(ko) 언어 설정
6. 응답에서 텍스트 추출
7. 임시 파일 삭제

# 템플릿 교체 함수 (replace_placeholders_in_docx)

주요 로직:
1. python-docx로 템플릿 파일 열기
2. 플레이스홀더 딕셔너리 구성
3. 문서의 모든 단락에서 플레이스홀더 찾기 및 교체
4. 테이블 내의 셀에서도 플레이스홀더 교체
5. 서식 (폰트, 크기, 굵기, 기울임) 유지

# 세션 상태 관리

다음 변수를 st.session_state에 저장:
- transcribed_text: STT 변환 결과
- generated_content: GPT가 생성한 회의 내용
- meeting_info: 사용자가 입력한 회의 정보 딕셔너리

# UI 레이아웃

세로 방향으로 순차 배치:
1. 📋 회의 정보 입력 폼
2. 🎤 음성 파일 업로드
3. 🔄 처리 버튼들
   - 1단계: STT 변환 → 결과 텍스트 수정 가능
   - 2단계: 회의록 생성 → 회의내용 수정 가능
   - 3단계: 최종 문서 생성 (DOCX/PDF 다운로드)

# 수정 가능 기능

- STT 변환 결과: AI가 잘못 인식한 단어 수정 가능
- 회의 내용: 생성된 회의록 내용 수정 가능

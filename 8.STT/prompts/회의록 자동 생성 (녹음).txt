# 역할
너는 실시간 녹음 기반 회의록 자동 생성 시스템을 만드는 풀스택 개발자다.
Streamlit 웹 앱을 구축하고, 실시간 녹음 후 OpenAI Whisper API로 STT 변환,
GPT로 회의록을 정리하여 Word 템플릿에 매핑하는 Python 프로그램을 작성한다.

# 현재 데이터 상태

이미 다음 파일이 존재한다:

- ./8.STT/회의록 양식.docx

템플릿 파일 구조:
- {{회의명}}: 회의 제목
- {{부서}}: 담당 부서
- {{작성자}}: 회의록 작성자
- {{회의일시}}: 회의 날짜 및 시간
- {{회의장소}}: 회의 장소
- {{회의안건}}: 회의 안건
- {{참석자}}: 참석자 명단
- {{회의내용}}: GPT가 생성한 회의 내용

- streamlit: 웹 앱 UI 구축

# 전체 작업 흐름

1. Streamlit 페이지 설정
   - 페이지 제목: "회의록 자동 생성 (녹음)"
   - 페이지 아이콘: 🎙️
   - 레이아웃: wide

2. 녹음 라이브러리 체크
   - audio-recorder-streamlit 패키지 설치 여부 확인
   - 미설치 시 설치 안내 메시지 표시

3. 환경 변수 설정
   - LLM 사용을 위한 API KEY는 .env에 있으니 반드시 python-dotenv의 load_dotenv()를 사용해 로드하세요.
   - .env 경로는 AI-Project/.env 입니다. 실행 위치와 무관하게 절대경로 기준으로 명시적으로 로드하세요.
   - 환경변수가 없으면 예외로 종료하지 말고 사용자에게 키 누락 메시지를 보여주세요.
   - API 키가 없으면 에러 메시지 표시

4. 템플릿 파일 경로 찾기
   - __file__ 기준으로 상위 폴더 검색
   - 현재 작업 디렉토리 기준 검색
   - 파일을 찾으면 성공 메시지, 못 찾으면 에러 메시지

5. 회의 정보 입력 폼
   - 회의명 (필수)
   - 작성자 (필수)
   - 회의 일시 (필수)
   - 회의 장소 (필수)
   - 부서 (필수)
   - 참석자 명단 (필수)
   - 회의 안건 (필수)

6. 실시간 녹음 섹션
   - audio_recorder() 함수로 녹음 버튼 표시
   - 마이크 아이콘 클릭으로 녹음 시작/종료
   - 녹음 완료 시 오디오 미리 재생 제공
   - 녹음 파일은 WAV 형식으로 저장
   - 녹음 파일 크기 표시

7. STT 변환 (1단계)
   - 녹음된 WAV 파일을 OpenAI Whisper API로 전송
   - 한국어(ko) 언어 설정
   - 변환된 텍스트 세션 상태에 저장
   - 사용자가 STT 결과를 직접 수정할 수 있도록 text_area 제공

8. 회의록 생성 (2단계)
   - GPT-4o-mini 모델 사용
   - JSON 형식으로 응답 받기 (response_format: json_object)
   - 회의내용만 생성
   - 불릿 리스트 형식으로 깔끔하게 정리

9. 최종 문서 생성 (3단계)
   - 템플릿의 플레이스홀더를 실제 데이터로 교체
   - DOCX 파일 다운로드 제공

# 녹음 기능 (audio_recorder)

from audio_recorder_streamlit import audio_recorder

audio_bytes = audio_recorder(
    text="",                    # 버튼 텍스트 (빈 문자열)
    recording_color="#e74c3c",  # 녹음 중 색상 (빨간색)
    neutral_color="#3498db",    # 대기 색상 (파란색)
    icon_name="microphone",     # 아이콘 이름
    icon_size="3x",             # 아이콘 크기
    pause_threshold=60.0,       # 무음 시 자동 종료 시간 (초)
    sample_rate=16000           # 샘플링 레이트
)

- 버튼 클릭: 녹음 시작
- 다시 클릭: 녹음 종료
- 반환값: bytes (WAV 형식)

# GPT 프롬프트 설계

## 시스템 프롬프트
"당신은 전문적인 회의록 작성자입니다. 반드시 JSON 형식으로 응답하고, 회의내용은 리스트가 아닌 하나의 문자열(줄바꿈 포함)로 작성하세요."

## 사용자 프롬프트 구조
- 회의 정보 (회의명, 회의안건, 참석인원)
- 변환된 음성 내용
- JSON 응답 형식 지정:
  {
      "회의내용": "반드시 하나의 문자열로 작성 (리스트 금지). 줄바꿈은 \\n 사용"
  }

## 중요 - 회의내용 형식
- GPT 응답의 "회의내용"은 반드시 문자열(string)이어야 함
- 리스트나 배열로 반환하면 안 됨
- 줄바꿈이 필요한 경우 \\n 사용
- 만약 리스트로 반환되면 format_content_to_string() 함수로 문자열 변환 처리

## 회의내용 작성 규칙
1. 회의 개요
   • 회의 목적과 배경 설명

2. 주요 논의 사항
   • 첫 번째 논의 사항
     - 세부 내용 및 의견
     - 결정 사항

3. 결정 사항
   • 결정된 사항 목록

4. 향후 계획
   • 후속 조치 사항

# STT 변환 함수 (transcribe_recorded_audio)

주요 로직:
1. 녹음된 bytes를 직접 API로 전송
2. requests로 OpenAI Whisper API 직접 호출
3. WAV 형식으로 전송 (Content-Type: audio/wav)
4. 한국어(ko) 언어 설정
5. 응답에서 텍스트 추출

# 세션 상태 관리

다음 변수를 st.session_state에 저장:
- transcribed_text: STT 변환 결과
- generated_content: GPT가 생성한 회의 내용
- recorded_audio: 녹음된 오디오 bytes
- meeting_info: 사용자가 입력한 회의 정보 딕셔너리

# UI 레이아웃

세로 방향으로 순차 배치:
1. 📋 회의 정보 입력 폼
2. 🎙️ 회의 녹음 (마이크 버튼)
3. 🔄 처리 버튼들
   - 1단계: STT 변환 → 결과 텍스트 수정 가능
   - 2단계: 회의록 생성 → 회의내용 수정 가능
   - 3단계: 최종 문서 생성 (DOCX/PDF 다운로드)

# 수정 가능 기능

- STT 변환 결과: AI가 잘못 인식한 단어 수정 가능
- 회의 내용: 생성된 회의록 내용 수정 가능

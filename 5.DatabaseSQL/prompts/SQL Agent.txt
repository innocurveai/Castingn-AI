# 역할
너는 Python 기반 B2B 지출 데이터 분석 대시보드 및 AI 지출 최적화 에이전트(AI Agent)를 구현하는 웹 솔루션 엔지니어다.
이미 구축이 완료된 로컬 SQLite DB(castingn_mro.db)를 연동하여, 평소에는 대화형 채팅창으로 동작하고 사용자가 원할 때만 DOCX 보고서를 다운로드하는 "Streamlit 시각화 어플리케이션(app.py)" 소스코드를 작성하라.

# 핵심 구현 목표 (★ 절대 인프라/DB 생성 코드를 짜지 말 것)
너는 오직 Streamlit 화면 UI 구성, Plotly 인터랙티브 차트 렌더링, st.chat_message 기반의 대화형 에이전트 인터페이스 화면을 만드는 데만 집중하라. (DB 테이블을 새로 만들거나 CSV를 파싱하여 인서트하는 코드는 완전히 제외한다.)

# 프로젝트 구조 및 경로
프로젝트 루트: ./5.DatabaseSQL/
- data/castingn_mro.db (이미 완벽히 데이터가 적재되어 있는 읽기 전용 SQLite DB)
- code/app.py (지금 네가 새로 작성해야 하는 Streamlit 시각화 소스코드)
- output/charts/ (보고서용 정적 PNG 파일 백업 저장 경로)
- output/reports/ (docx 결과물 저장 경로)

# 데이터베이스 연결 및 테이블 정보
- 연결 라이브러리: sqlite3
- 대상 테이블: mro_expenses
- 컬럼 구조: month, company_name, category, item, spec, channel, price_source, quantity, price
※ 프로그램 구동 시 'sqlite3.connect("./5.DatabaseSQL/data/castingn_mro.db")' 또는 'data/castingn_mro.db' 경로를 바인딩하여 SELECT 쿼리로 데이터를 로드하라.

# 필수 UI 및 시각화 기능 요구사항

1. 상단 타이틀 및 KPI 카드 영역
   - 타이틀: "📊 캐스팅엔 MRO 지출 혁신 및 비용 절감 분석 시스템"
   - st.metric을 가로로 배치하여 '총 지출액', '업무마켓9 도입 총 절감액', '평균 비용 절감률(%)'을 연산하여 직관적인 카드로 노출하라.

2. 중앙 대화형 분석 영역 (채팅 UI)
   - st.chat_message와 st.chat_input을 사용하여 챗봇 화면을 빌드하라.
   - 사용자가 질문을 던지면 내부적으로 LLM이 SQL 변환 및 실행을 거쳐 결과를 '일반 문단 텍스트'와 '불렛 리스트' 형태로만 답변하도록 프롬프팅하라. (서식 꼬임 방지를 위해 #, **, ` 등 마크다운 기호 금지)
   - 질문 의도에 따라 화면에 Plotly 기반 꺾은선 차트(월별 추이) 또는 막대 차트(채널별 단가 비교)를 즉각 렌더링하라.

3. 하단 리포트 생성 섹션 (선택형 보고서 파이프라인)
   - 에이전트 답변 하단에 "BSM 비용 절감 제안 리포트 생성" 버튼을 노출하라.
   - 사용자가 클릭 시, 화면에 그려진 Plotly 차트를 정적 PNG로 변환하여 output/charts/에 임시 저장하고, python-docx 라이브러리를 사용해 표지, 데이터 요약, 차트 이미지가 바이너리로 포함된 완벽한 Word 문서를 완성하여 output/reports/에 빌드 후 다운로드 버튼(st.download_button)을 활성화하라.

4. 사이드바 제어 패널
   - 모델 선택 컴포넌트, OpenAI 실시간 웹 검색(web_search) 온오프 체크박스, 대화 초기화 버튼을 명확히 구현하라.

5. 환경 변수 로드 안전 규칙
   - AI-Project/.env의 절대경로 기준으로 load_dotenv()를 수행하되, API Key가 누락되었을 경우 프로그램이 멈추지(Crash) 않도록 st.error를 통해 화면에 예쁘게 경고 창을 띄워라.

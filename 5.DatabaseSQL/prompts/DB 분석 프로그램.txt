# 역할

너는 기업 지출 관리 플랫폼(업무마켓9)의 데이터를 분석하는 데이터 분석가이자,
Python 기반 데이터 시각화 및 Streamlit 분석 대시보드를 구축하는 엔지니어다.

castingn_mro.db SQLite 데이터베이스를 기반으로,
고객사들의 MRO/비품 지출 현황을 탐색하고 일반 유통 채널 대비 업무마켓9의 비용 절감 효과를 한눈에 증명할 수 있는
"캐스팅엔 BSM(Business Spend Management) 비용 분석 대시보드 프로그램"을 작성한다.

# 프로젝트 구조

프로젝트 루트: ./5.DatabaseSQL/

- data/
  - castingn_mro.db
- code/
  - (여기에 streamlit 분석 프로그램 생성)

# 데이터베이스 정보

파일:
./5.DatabaseSQL/data/castingn_mro.db

테이블:
mro_expenses

컬럼 구조:
- month (text) -> 기준월 (YYYY-MM)
- company_name (text) -> 고객사명
- category (text) -> 구매 카테고리 (사무용품, IT자산, 생활탕비 등)
- item (text) -> 품목명
- spec (text) -> 규격
- channel (text) -> 구매채널 (일반유통채널, 업무마켓9)
- price_source (text) -> 가격출처 (시중평균가, 캐스팅엔제휴가 등)
- quantity (integer) -> 구매수량
- price (real) -> 단가 (1개당 가격)

# 프로그램 목표

Streamlit 기반 
"캐스팅엔 MRO 지출 혁신 및 비용 절감 분석 시스템" 구축

이 프로그램은 단순 시각화가 아니라:
- 기업별/카테고리별 지출 규모 탐색 (EDA)
- 일반 채널 대비 업무마켓9의 단가 비교 및 총 절감액(ROI) 추적
- 구매 수량과 단가 변동에 따른 지출 추세 분석
- 대량 구매 및 정기 구독 효율성 검증
이 가능한 “기업 맞춤형 지출 최적화 분석 도구”여야 한다.

# 필수 기능

## 1. 데이터베이스 연결
- sqlite3 또는 SQLAlchemy를 사용하여 로컬의 castingn_mro.db 연결
- mro_expenses 테이블 로드 후, 데이터 존재 여부와 전체 row 수 출력
- DB 연결 오류 발생 시 사용자가 직관적으로 알 수 있는 경고 메시지(st.error) 표시

## 2. 사이드바 필터 시스템
다음 조건을 Streamlit sidebar에서 실시간 인터랙티브하게 선택 가능하도록 구성:
- 기준월(month) : 멀티 선택 (기본값: 전체 선택)
- 고객사명(company_name) : 멀티 선택 또는 단일 선택
- 카테고리(category) : 멀티 선택
- 품목명(item) : 텍스트 검색 및 선택
- 단가 범위(price) : 최소값~최대값 슬라이더 (st.slider)
필터 변경 시 대시보드의 모든 KPI와 차트가 실시간 동기화되어 반영되도록 구성

## 3. 데이터 탐색 및 요약 영역
- 사용자가 필터링한 최종 DataFrame 테이블을 화면에 명확히 표시 (st.dataframe)
- 현재 선택된 필터 조건 요약(총 고객사 수, 대상 품목 수, 총 구매 수량)을 가독성 있게 출력
- 데이터의 스키마 구조와 비즈니스적 의미를 간략히 설명하는 Help 컴포넌트 제공

## 4. 핵심 비즈니스 KPI 지표 영역 (카드 형태)
Streamlit의 `st.metric`을 사용하여 상단에 가로로 배치:
- **총 지출 금액**: 현재 필터링된 데이터의 총 구매 금액 (단가 * 수량의 총합)
- **업무마켓9 도입 총 절감액**: (일반유통채널 단가 - 업무마켓9 단가) * 구매수량 공식으로 계산된 순수 절감 비용
- **평균 비용 절감률(%)**: 일반 채널 대비 업무마켓9의 평균 단가 할인 비율
- **최다 지출 카테고리**: 가장 높은 지출 비중을 차지하는 카테고리명 및 금액

## 5. 분석 목적별 시각화 섹션 (Plotly 또는 Altair 활용)
그래프는 확대, 마우스 오버 툴팁, 범례 제어가 가능하도록 구현한다.

### (1) 월별 지출 및 절감 추세 분석 (Line / Area Chart)
- x축 = month, y축 = 총 지출액 또는 총 절감액
- 그룹(Color) = channel 또는 category 선택형 구성
- 목적: 도입 개월 수가 지나감에 따라 기업의 지출이 얼마나 안정화되고 절감액이 커지는지 추세 분석

### (2) 구매 채널별 단가 비교 분석 (Grouped Bar Chart)
- 동일 품목(`item`) 및 규격(`spec`)에 대해 '일반유통채널'과 '업무마켓9'의 단가를 나란히 비교하는 막대그래프
- 목적: 캐스팅엔 플랫폼의 가격 경쟁력을 품목별로 명확하게 시각화

### (3) 카테고리별 지출 비중 및 분포 (Pie / Box Plot)
- 카테고리별 전체 지출 총액 비중(Pie) 및 각 채널별 단가 변동 폭과 이상치 탐색(Box Plot)
- 목적: 어느 부서/카테고리에서 비효율적인 지출(이상치)이 발생하고 있는지 모니터링

### (4) 지출 구조 패턴 분석 (Heatmap)
- x축 = month, y축 = company_name 또는 category
- 내부 값 = 총 지출액 혹은 평균 절감율
- 목적: 특정 시기(예: 연말 소모품 집중 구매기)나 특정 기업의 집중 지출 패턴과 절감 타이밍 포착

### (5) 비용 절감 기여도 Top-N 품목 탐색 (Horizontal Bar Chart)
- 업무마켓9을 통해 가장 많은 비용을 아낀 품목 TOP 5 / TOP 10 시각화
- 목적: 고객사 재계약 및 영업 브리핑 시 가장 효과적인 레퍼런스 데이터 제시

## 6. AI 기반 비용 혁신 자동 요약 리포트 (Text Generation)
통계 수치 연산 결과를 조합하여 동적으로 변하는 텍스트 리포트 영역 구현
(예시: "2026년 상반기 분석 결과, (주)에이비씨를 포함한 주요 고객사들은 '업무마켓9' 도입을 통해 총 OOO원의 지출을 기록했으며, 시중가 대비 평균 OO%의 비용 절감 혁신을 달성했습니다. 특히 'IT자산(모니터 등)' 카테고리에서 가장 높은 절감 효율을 보였습니다.")

## 7. 코드 구조 및 품질 요구사항
- 전 과정 함수화 설계 (load_data, apply_filters, calculate_kpi, draw_charts..., main)
- @st.cache_data를 활용한 SQLite 조회 속도 최적화 적용
- 한글 인코딩 및 결측치 데이터 처리 시 예외 처리(try-except) 필수 적용
- 코드 상단에 프로젝트 아키텍처 및 실행 명령어 주석 명시
